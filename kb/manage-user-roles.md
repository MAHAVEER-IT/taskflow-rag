# Manage User Roles

## Overview

User roles control what each person can see and do in TaskFlow. Roles help protect workspace settings, billing information, project access, files, and team membership. The standard workspace roles are **Admin**, **Manager**, and **Member**.

Admins manage the workspace. Managers coordinate projects and team activity. Members complete assigned work, comment on tasks, and collaborate in projects they can access.

## Steps

1. Open your TaskFlow workspace.
2. Select **Workspace settings**.
3. Select **Members**.
4. Find the user you want to update.
5. Open the role menu next to the user’s name.
6. Choose **Admin**, **Manager**, or **Member**.
7. Review the permission summary.
8. Select **Save changes**.

Admins can also adjust project-level access by opening a project, selecting **Project settings > Members**, and adding or removing users. Project-level access works together with workspace roles. For example, a Member can be added to a private project without receiving full workspace management permissions.

## Important Notes

- Admins can manage billing, subscription plans, workspace settings, members, roles, integrations, and all projects.
- Managers can create and manage projects, invite Members when allowed, assign tasks, and manage project files.
- Members can view assigned projects, update tasks, comment, and upload files where they have access.
- At least one Admin must remain in every workspace.
- Only Admins can promote another user to Admin.
- Role changes take effect immediately and may change what the user sees after refreshing TaskFlow.

## Troubleshooting

If you cannot change a user’s role, you may not be an Admin. Ask a current Admin to make the change.

If the role menu is disabled, the user may be the only remaining Admin. Promote another trusted user to Admin before changing or removing that account.

If a user still cannot access a project after a role update, check the project visibility and project member list. Private projects require explicit access.

If someone has more access than expected, review both their workspace role and project-level membership.

## FAQ

**What is the difference between a Manager and an Admin?**  
Managers focus on project work. Admins can also manage billing, workspace settings, roles, and subscription-level controls.

**Can a Member upload files?**  
Yes, if they have access to the project or task where the file is being uploaded.

**Can I create custom roles?**  
Custom roles are available on selected TaskFlow plans. Admins can check **Workspace settings > Roles**.

**Does removing a user delete their work?**  
No. Their tasks, comments, and uploaded files remain in the workspace history.
