# Create Project

## Overview

Projects organize your team’s tasks, files, comments, and deadlines in TaskFlow. A project can represent a client engagement, product launch, internal initiative, marketing campaign, or any other body of work.

Admins and Managers can create projects by default. Members may also create projects if an Admin has enabled that permission in workspace settings.

## Steps

1. Open your TaskFlow workspace.
2. Select **Projects** in the sidebar.
3. Choose **New project**.
4. Enter a project name.
5. Add an optional project description.
6. Choose a project owner. The owner is responsible for project settings and coordination.
7. Select a visibility option: **Workspace visible** or **Private**.
8. Set optional start and due dates.
9. Choose a project template, or select **Blank project**.
10. Select **Create project**.

After the project is created, add task lists, assign tasks, upload files, and invite team members. You can update project settings by opening the project and selecting **Project settings**.

To archive a project, open **Project settings**, select **Archive project**, and confirm. Archived projects become read-only and move out of the active project list.

## Important Notes

- Workspace-visible projects can be found by all workspace members.
- Private projects are visible only to invited project members, Managers with access, and Admins.
- Archiving a project does not delete tasks, comments, files, or history.
- Archived projects can be restored by an Admin or the project owner.
- Project names should be clear and specific so teammates can find them in search.
- Billing limits may apply to the number of active projects available on your plan.

## Troubleshooting

If you cannot create a project, your role may not include project creation permission. Ask an Admin to update your role or create the project for you.

If a teammate cannot see the project, check the project visibility setting. For private projects, add the teammate from **Project settings > Members**.

If the project template does not appear, it may be available only on a higher subscription plan or restricted by your workspace Admin.

If you archived the wrong project, open **Projects > Archived**, select the project, and choose **Restore project**.

## FAQ

**Can I rename a project after creating it?**  
Yes. Open **Project settings**, edit the project name, and save your changes.

**Can I change a project from private to workspace visible?**  
Yes, if you are an Admin or the project owner. Open **Project settings > Visibility**.

**Do archived projects count toward active project limits?**  
No. Archived projects are stored for reference but do not count as active projects.

**Can I copy an existing project?**  
Yes. Use **Duplicate project** from the project menu to copy task lists, settings, and optional tasks.
